# Compatibility Guide

This document explains compatibility expectations for **BetbetMiro Extension**.

It covers CloudStream app behavior, repository metadata, Gradle builds, GitHub Actions, and provider runtime validation.

---

## Compatibility Scope

Compatibility in this repository means:

- The extension repository can be added to CloudStream.
- `repo.json` points to a valid plugin list.
- `plugins.json` is generated and reachable from the expected branch/path.
- Provider modules compile.
- Provider cards, details, and playback flows work in CloudStream when the source is alive.

A provider can compile successfully but still be broken at runtime. Build compatibility and runtime compatibility are related, but they are not the same thing.

---

## CloudStream Stable vs Prerelease

CloudStream behavior can differ between stable and prerelease builds.

When reporting or validating a provider, always mention which channel was used:

- Stable
- Prerelease
- Beta or custom build
- Unknown

A provider issue should not be assumed to be source-side until it is checked against the app version/channel used by the reporter.

---

## Provider Runtime Compatibility

A provider should be considered runtime-compatible only when the relevant CloudStream flow works.

Recommended checks:

1. Homepage or main page loads.
2. Categories contain usable cards.
3. Search works when supported by the source.
4. Detail page opens without crashing.
5. Episode list or movie play item is available.
6. `loadLinks()` emits at least one video callback link when playback is supported.
7. Subtitles load when supported and available.

Do not treat parser compilation as proof of runtime compatibility.

---

## Playback Compatibility

Playback is the most sensitive part of a provider.

A playback flow may require:

- Referer header.
- Origin header.
- Cookies.
- User agent.
- Iframe extraction.
- API token extraction.
- JavaScript-packed source decoding.
- Media-player script parsing.
- HLS `.m3u8` or direct `.mp4` handling.
- Subtitle extraction.

A `loadLinks()` function returning `true` does not prove playback. A working resolver must emit at least one video callback link.

If runtime playback was not tested directly, say so clearly.

Recommended wording:

```text
Playback belum terbukti di runtime/device, tetapi resolver sudah disesuaikan dengan evidence source.
```

If HAR/log evidence was used:

```text
Playback belum terbukti di runtime/device, tetapi resolver sudah disesuaikan dengan evidence source/HAR.
```

## Gradle / Java Compatibility

Use the repository Gradle wrapper whenever possible:

```bash
./gradlew make
```

For Windows:

```bat
.\gradlew.bat make
```

Do not rely on a globally installed Gradle version unless needed for debugging.

For Java, use the version expected by the current Gradle/Android/Kotlin toolchain. If a build fails, include the Java version in the issue report.

Useful command:

```bash
java -version
```

---

## Repository Metadata Compatibility

The repository metadata should remain valid:

- `repo.json` should use the expected CloudStream repository format.
- `repo.json` should point to the correct plugin list location.
- `plugins.json` should be generated by the build workflow.
- Plugin artifacts should be available from the expected release/build output.

When changing build or workflow files, check whether `repo.json` and `plugins.json` are affected.

---

## GitHub Actions Compatibility

GitHub Actions should be treated as the remote build authority for repository publishing.

A local build can pass while Actions fails because of:

- Java version mismatch.
- Gradle cache differences.
- Missing permissions.
- Branch/output path differences.
- Artifact upload or plugin list generation issues.

Do not claim Actions success unless the workflow run was verified.

---

## Android / Device Compatibility

CloudStream provider behavior may differ by device or Android environment.

Relevant details for reports:

- Android phone.
- Android tablet.
- Android TV.
- Emulator.
- CloudStream stable/prerelease/custom.
- VPN/DNS/region.
- Source website accessibility in browser.

Some source websites behave differently depending on region, DNS, Cloudflare, user agent, or cookie state.

---

## Source Website Compatibility

Third-party sources can change without notice.

Common breaking changes:

- Domain redirect.
- Cloudflare or anti-bot protection.
- HTML layout changes.
- Search endpoint changes.
- Player iframe changes.
- Host/server list changes.
- API token or script changes.
- Subtitle path changes.
- Region blocking.

Provider fixes should follow active source evidence, not assumptions from old parser behavior.

---

## Domain Compatibility

When a source changes domain:

1. Verify the old domain behavior.
2. Verify the new domain is reachable.
3. Confirm homepage/category behavior.
4. Confirm detail/load behavior.
5. Confirm playback resolver behavior.
6. Update `mainUrl` only after evidence is clear.
7. Bump provider version if provider code changes.

Domain redirect alone does not prove full provider compatibility.

---

## Issue Report Compatibility Checklist

When reporting compatibility problems, include:

- Provider name or affected repository area.
- CloudStream version/channel.
- Device type.
- Android version when relevant.
- Source URL.
- Steps to reproduce.
- Logs or screenshots when available.
- Whether the source website opens in a browser.
- Whether VPN/DNS/region affects the result.

For build issues, also include:

- OS.
- Java version.
- Gradle command used.
- Full relevant error output.

---

## Maintainer Rule

Compatibility claims should be honest and evidence-backed.

Use clear status labels:

```text
Gradle build lokal: SUCCESS / gagal / belum dijalankan
repo.json: valid / invalid / belum dicek
plugins.json: valid / invalid / belum dicek
GitHub Actions: sukses / gagal / belum dijalankan / belum diverifikasi
Homepage category cards: terbukti / belum terbukti
Load detail/episode: terbukti / belum terbukti
Playback callback link > 0: terbukti / belum terbukti
```

Do not claim a status that was not actually verified.
